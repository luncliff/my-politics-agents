# 성남시 조례안 작성 시작 가이드

이 문서는 성남시의회 제출을 염두에 두고 조례안을 처음 잡는 사람을 위한 실무형 안내서다. `docs/ordinance-drafting/ordinance-package-template.md`를 바로 채우기 전에, 무엇을 먼저 판단하고 어떤 순서로 초안을 만들지 정리하는 데 목적이 있다.

법제처 국민참여입법센터의 자치법규 입안 길라잡이는 입안 형식, 조문 체계, 표현기준의 상위 원칙을 제공하고, 행정안전부 자치법규 입법 절차 자료는 입법예고, 법률검토, 비용추계, 공포까지의 절차 축을 제공한다. 경기도교육청 자치법규 입안 실무 매뉴얼 계열 자료는 초안 작성 순서, 작성요령, 실례 중심의 실무 감각을 보완하는 기준으로 본다.

## 1. 이 가이드는 언제 쓰는가

- 새 정책이나 제도를 성남시 차원에서 제도화하려고 할 때
- 기존 조례를 일부 고치거나 전면 재설계하려고 할 때
- 실효성이 없어진 조례를 폐지하려고 할 때
- 의원발의안, 위원회안, 시장 제출안을 검토하거나 초안을 대신 정리해야 할 때

아래 네 경우가 아니면 조례안이 아니라 규칙, 지침, 내부계획, 예산사업 설명자료로 처리하는 것이 맞을 수 있다.

- 시민의 권리와 의무, 지원 근거, 위원회 설치, 비용 지원, 행정상 의무를 규정해야 한다.
- 상위법이 조례 위임을 하고 있다.
- 집행부가 예산과 조직을 동원해 계속 집행해야 한다.
- 의회 의결을 거친 규범 형식이 필요하다.

## 2. 시작 전에 먼저 판단할 것

### 2.1 조례가 맞는지부터 본다

- 상위법이 이미 시행령, 시행규칙, 중앙행정기관 고시로 충분히 규율하고 있으면 조례로 반복하지 않는다.
- 내부 사무처리 기준이나 부서 간 업무분장은 조례보다 규칙이나 지침이 더 적합할 수 있다.
- 선언만 있고 집행수단이 없는 문장은 조례 본문보다 제안이유나 정책설명서에 두는 편이 낫다.

### 2.2 제정인지, 일부개정인지, 전부개정인지, 폐지인지 정한다

- `제정`: 새 제도를 처음 만든다.
- `일부개정`: 현행 조례의 구조는 유지하되 조문 일부만 바꾼다.
- `전부개정`: 현행 조례의 기본 틀을 사실상 다시 짠다.
- `폐지`: 제도의 필요가 사라졌거나 다른 조례로 흡수된다.

판단 기준은 간단하다. 조문 번호를 조금 고치는 수준이면 일부개정, 목적 조항부터 집행체계까지 다시 짜야 하면 전부개정으로 본다.

### 2.3 상위법 근거와 충돌 여부를 확인한다

- 근거 법률, 시행령, 시행규칙이 있는지
- 상위법이 조례에 위임한 범위 안에 들어오는지
- 성남시 기존 조례와 중복되거나 충돌하지 않는지
- 벌칙, 과태료, 부담금, 사용료처럼 강한 규율은 상위법 근거가 분명한지

상위법 근거가 흐리면 초안을 길게 쓰기 전에 먼저 여기서 멈추는 편이 맞다.

### 2.4 재정과 집행주체를 먼저 적어본다

- 돈이 드는가
- 어느 부서가 집행하는가
- 지원 대상, 신청 요건, 심사 방식이 잡혀 있는가
- 위원회가 필요한가, 아니면 기존 위원회로 처리 가능한가

이 네 가지가 비어 있으면 조례 문장은 써도 실제 집행 단계에서 막힌다.

## 3. 가장 먼저 만드는 초안 순서

처음부터 신구조문대비표를 만들지 말고, 아래 순서로 본문 골격부터 잡는다.

### 3.1 제목

- 제정안: `성남시 OOO 조례안`
- 일부개정안: `성남시 OOO 조례 일부개정조례안`
- 전부개정안: `성남시 OOO 조례 전부개정조례안`
- 폐지안: `성남시 OOO 조례 폐지조례안`

제목만 보고도 행위 유형이 드러나야 한다.

### 3.2 제1조 목적

목적조항은 정책 슬로건이 아니라 이 조례가 무엇을 위해 무엇을 규율하는지 쓰는 자리다.

- 왜 필요한지
- 무엇을 하려는지
- 누가 적용대상인지

세 문장이 한 번에 다 들어가도 되지만, 의미는 하나로 유지한다.

### 3.3 정의

정의 조항은 꼭 필요한 용어만 둔다.

- 지원대상
- 전담부서
- 사업
- 시설
- 위원회

일상어로 충분한 단어까지 억지로 정의하면 읽기만 어려워진다.

### 3.4 적용 범위와 대상

누가 지원을 받는지, 어떤 사무에 적용되는지, 예외는 무엇인지 먼저 적는다. 지원대상과 적용범위가 흐리면 뒤 조문이 모두 흔들린다.

### 3.5 책무와 권한

통상 아래 항목 중 필요한 것만 둔다.

- 시장의 책무
- 시민 또는 사업수행자의 협력 의무
- 주관부서의 시행 권한
- 계획 수립 또는 실태조사 권한

책무 조항은 선언으로 끝내지 말고 뒤 조문과 연결되게 쓴다.

### 3.6 핵심 집행수단

정책성 조례는 보통 아래 조문군 중 일부가 반복된다.

- 기본계획 또는 시행계획
- 지원사업
- 신청, 선정, 취소
- 위탁 또는 협력
- 교육, 홍보, 실태조사
- 위원회 심의
- 포상, 자료제출, 보고

필요한 제도만 넣고, 실제 집행하지 않을 장식 조문은 뺀다.

### 3.7 부칙

최소한 아래를 판단한다.

- 시행일
- 경과조치 필요 여부
- 다른 조례 개정 필요 여부
- 종전 규정에 따른 처분의 효력 정리 여부

부칙이 약하면 시행 단계에서 해석 다툼이 생긴다.

## 4. 조문을 쓸 때 지켜야 할 기본 원칙

### 4.1 한 문장에 한 뜻만 둔다

- 한 조문에 목적, 대상, 절차, 예외를 다 몰아넣지 않는다.
- 예외가 길어지면 항이나 호로 나눈다.
- 문장이 길어질수록 해석 여지가 커진다.

### 4.2 용어를 끝까지 같게 쓴다

- `지원대상자`, `수혜자`, `신청인`을 섞지 않는다.
- `시장`, `성남시장`, `시장은`을 문맥에 맞게 통일한다.
- 같은 뜻의 단어를 바꿔 쓰는 것은 문학에서는 장점이지만 조례에서는 단점이다.

### 4.3 상위법 문장을 그대로 반복하지 않는다

- 상위법이 이미 규정한 일반원칙을 조례에서 다시 선언만 하면 실익이 작다.
- 조례에는 성남시가 실제로 정해야 하는 범위만 남긴다.
- 상위법 위임 없이 과도한 의무, 제재, 제한을 추가하지 않는다.

### 4.4 조문 표제는 기능이 드러나게 쓴다

- `제1조(목적)`
- `제2조(정의)`
- `제5조(지원사업)`
- `제8조(위원회의 설치 및 기능)`

`기타`, `운영사항`, `필요한 사항`처럼 뭉툭한 표제는 가능한 줄인다.

### 4.5 위임 조항은 좁고 분명하게 쓴다

- `필요한 사항은 규칙으로 정한다`만 남발하지 않는다.
- 어떤 사항을 규칙으로 정할지 범위를 적는다.
- 본문에서 정해야 할 핵심까지 규칙으로 넘기지 않는다.

### 4.6 개정안은 바뀌는 포인트가 바로 보이게 쓴다

- 일부개정안은 현행 조례 어디를 왜 고치는지 제안이유와 대비표에서 바로 보여야 한다.
- 전부개정안은 기존 조례 체계가 왜 더 이상 유지되기 어려운지 먼저 설명해야 한다.
- 폐지안은 대체 제도, 실효 상실, 중복 규율 중 어느 이유인지 명확히 적는다.

## 5. 조례안 패키지 준비물

`docs/ordinance-drafting/ordinance-package-template.md`는 아래 묶음을 한 번에 작성하도록 설계되어 있다.

- `표지 메타`: 조례안 유형, 발의 구분, 소관위원회, 상위 근거법령, 관련 조례, 검토 상태
- `조례안 본문`: 제안하려는 규범 자체
- `제안이유`: 배경, 법적 근거, 주요 내용, 기대 효과
- `신구조문대비표`: 일부개정 또는 전부개정에서 필수
- `비용추계 또는 비용추계 비대상 사유`: 돈이 드는지, 왜 안 드는지
- `선택 첨부물`: 관계법령 발췌서, 의견수렴 메모, 수정가결 내용 등
- `체크리스트`: 제출 전에 빠뜨린 항목이 없는지 확인

실무적으로는 본문만 써 놓고 나머지를 비워 두는 경우가 가장 위험하다. 실제 심사에서는 제안이유, 대비표, 비용추계가 빠진 초안이 가장 먼저 약점으로 드러난다.

## 6. 제안 전에 반드시 하는 자가점검

- [ ] 상위법 근거 조문을 바로 제시할 수 있는가
- [ ] 현행 성남시 조례와 중복되거나 충돌하는 조문이 없는가
- [ ] 지원대상, 적용범위, 집행부서가 문장으로 분명한가
- [ ] 실태조사, 계획수립, 위원회, 위탁 같은 수단이 실제 집행 가능한가
- [ ] 비용이 들면 재원과 집행방식이 설명되는가
- [ ] 일부개정이면 무엇이 어떻게 바뀌는지 대비표만 봐도 드러나는가
- [ ] 폐지안이면 대체 제도 또는 폐지 필요성이 충분히 설명되는가
- [ ] 부칙에서 시행일, 경과조치, 다른 조례 정리가 필요한지 검토했는가
- [ ] 조문 표제와 용어가 문서 전체에서 통일되어 있는가

이 체크리스트에서 두세 개 이상 막히면 조문을 더 쓰기보다 제안 구조를 다시 잡는 편이 낫다.

## 7. 성남시의회 제출 흐름에서 볼 포인트

성남시의회 실무 흐름에 맞추면 초안 작성자는 최소한 아래 단계를 의식해야 한다.

1. 발의 또는 제출 준비
2. 법률검토와 내부 협의
3. 입법예고 또는 의견수렴
4. 비용추계, 관계부서 검토, 필요한 경우 규제 검토
5. 상임위원회 심사
6. 본회의 의결
7. 이송 및 공포

즉, 조례안 본문은 첫 단계일 뿐이고, 제안이유와 비용추계, 관계법령, 의견수렴 흔적이 뒤에서 같이 따라와야 한다.

## 8. 템플릿을 실제로 쓰는 순서

### 8.1 첫날에는 본문보다 표지 메타를 먼저 채운다

아래 여섯 칸이 비어 있으면 아직 초안을 쓸 준비가 덜 된 상태다.

- 조례안 유형
- 발의 구분
- 상위 근거법령
- 관련 기존 자치법규
- 소관위원회
- 대표발의자 또는 제출자

### 8.2 둘째로 본문 골격만 잡는다

`목적 -> 정의 -> 적용범위 -> 책무 -> 집행수단 -> 부칙` 순서로 빈칸 초안을 만든다. 처음부터 문장을 다듬기보다 어떤 제도가 필요한지부터 배치한다.

### 8.3 셋째로 제안이유와 비용추계를 쓴다

본문을 다 쓴 뒤 제안이유를 쓰면, 왜 이 조례가 필요한지 논리가 흔들리는 부분이 잘 보인다. 비용이 없다면 왜 없는지 근거를 적는다.

### 8.4 마지막에 비교표와 첨부물을 붙인다

일부개정안은 신구조문대비표가 사실상 설명서 역할을 한다. 심사 과정에서 수정이 붙을 가능성이 있으면 수정 포인트를 별도 메모로 정리해 두는 편이 낫다.

  - 본문 HWP: `archive/raw/www.sncouncil.go.kr/manual-bill-5924-attach-01-5924-AI반도체과---성남시-모빌리티-혁신-및-활성화-지원-조례안(이군수-의원-등-12명).hwp`

예시는 표현을 베끼기 위한 것이 아니라, 어떤 항목이 실제로 패키지에 들어가는지 보는 용도로 사용한다. 현재 확보된 2024~2025 성남시의회 표본은 주로 HWP/HWPX 형식이고, 스크립트는 의회가 PDF를 제공하는 경우 PDF도 같은 방식으로 저장한다.

## 10. 스크립트 사용법

`scripts/collect_seongnam_ordinance_samples.py`는 두 가지 모드로 쓴다.

### 10.1 반기 표본 수집

반기별 표본 2건씩 다시 만들거나 갱신할 때는 `--window`를 쓴다.

```powershell
python scripts\collect_seongnam_ordinance_samples.py --window 2025-1H --overwrite
```

산출물은 아래에 쌓인다.

- 표본 메모: `archive/processed/ordinance-drafting/samples/<window>/<committee>/<bill_no>.md`
- 반기 요약: `archive/processed/ordinance-drafting/samples/<window>/SUMMARY.md`
- 원문 HTML/HWP/HWPX/PDF: `archive/raw/www.sncouncil.go.kr/<window>-bill-<bill_no>*`

### 10.2 조례안 번호로 직접 다운로드

특정 조례안을 실무 참고용으로 바로 열어보고 싶을 때는 `--bill-no`를 쓴다. 이 모드는 의안 상세 HTML과 첨부 HWP/HWPX/PDF를 `manual-bill-*` 이름으로 따로 저장하고, 참고 메모도 함께 만든다.

```powershell
python scripts\collect_seongnam_ordinance_samples.py --bill-no 5609
```

여러 번호를 한 번에 받을 수도 있고, 쉼표로도 넣을 수 있다.

```powershell
python scripts\collect_seongnam_ordinance_samples.py --bill-no 5637,5784 --window 2024-2H --window 2025-1H
```

첨부 형식을 제한하고 싶으면 `--attachment-ext`를 반복한다. 예를 들어 HWP와 PDF만 남기고 싶으면 아래처럼 쓴다.

```powershell
python scripts\collect_seongnam_ordinance_samples.py --bill-no 5924 --attachment-ext hwp --attachment-ext pdf
```

`--bill-no` 모드 산출물은 아래에 쌓인다.

- 상세 HTML: `archive/raw/www.sncouncil.go.kr/manual-bill-<bill_no>.html`
- 첨부 HWP/HWPX/PDF: `archive/raw/www.sncouncil.go.kr/manual-bill-<bill_no>-attach-*`
- 참고 메모: `archive/processed/ordinance-drafting/references/<bill_no>.md`

이미 반기 표본 수집에서 같은 의안을 받아 둔 경우에는 기존 raw 파일을 재사용해 `manual-bill-*` 번들을 만들고, `--overwrite`를 주면 다시 내려받는다.

## 11. 빠르게 시작하는 최소 작업 순서

시간이 없으면 아래 순서만 지켜도 초안 품질이 크게 떨어지지 않는다.

1. 상위법 근거 1개와 현행 조례 1개를 먼저 적는다.
2. 제목과 조례안 유형을 확정한다.
3. 목적, 정의, 적용범위, 핵심 집행수단, 부칙만 먼저 쓴다.
4. 제안이유 4문단과 비용추계 또는 비대상 사유를 채운다.
5. 일부개정안이면 신구조문대비표를 붙인다.
6. 제출 전 체크리스트를 한 번 돌린다.
