# 세션 회고 작성

이번 세션의 작업 내용을 정리하고 `회고/` 폴더에 저장한다.

## 절차

1. 이번 세션에서 시도한 것, 성공한 것, 막힌 것을 대화 맥락에서 추출한다.
2. 새로 발견한 사이트·포맷·정책을 목록화한다.
3. 같은 작업이 두 번 이상 반복되었으면 **자동화 후보**로 분류한다:
   - skill 후보: 단일 목적 반복 작업
   - agent 후보: 도메인 전문 페르소나 필요 시
   - task 후보: VS Code 태스크로 자동화 가능한 것
   - hook 후보: Claude Code 라이프사이클 이벤트로 처리 가능한 것
4. 출처·PII 점검: 누락된 출처나 마스킹되지 않은 PII가 있으면 즉시 수정 후 기록.
5. 파일명: `YYYY-MM-DD <slug>.md` (한국어 slug 가능) — 오늘 날짜 기준.
6. `회고/` 아래에 저장.

## 출력 템플릿

```markdown
---
date: YYYY-MM-DD
slug: <slug>
duration_min: <대략>
---

## 시도한 것

- ...

## 성공한 것

- ...

## 막힌 것 / 다음에 해결

- ...

## 새로 알게 된 사이트·포맷·정책

- ...

## 자동화 후보

- skill: ...
- agent: ...
- task: ...
- hook: ...

## 출처·PII 점검 결과

- 누락 N건 / 조치: ...
```

## 시간이 부족할 때

최소한 **"다음에 자동화할 후보"** 한 줄만이라도 남긴다.
