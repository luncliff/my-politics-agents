# local-ordinance-processor

대한민국 지방자치단체 조례·규칙·자치법규를 최신성, 정확성, 재현성을 유지한 상태로 수집하고 주제별로 누적 저장한다.

## 언제 쓰는가

- 특정 지자체의 최신 조례 목록을 확인할 때
- 조례안이 실제 공포·시행된 조례인지 검증할 때
- ELIS, 지방의회, law.go.kr의 정보를 교차 확인해야 할 때
- 행정 부서 구조가 아니라 주제별(Semantic) 구조로 재정리할 때
- 기존 조례 Markdown에 개정 이력을 append 할 때

## 대상 지자체 식별

- 작업 시작 시 저장소 루트의 [`location.txt`](../../../location.txt)를 먼저 읽어 **대상 행정구역**을 확정한다.
- 본 스킬에서 `<지자체>`는 기초/광역 자치단체 이름을 가리킨다.
- 사용자가 채팅에서 다른 지자체를 명시하면 그 지시가 `location.txt`보다 우선한다.

## 입력과 산출물

- 입력 예시: `성남시 최신 조례 목록`, `성남시 도시계획 조례`, `성남시 보건복지 조례 정리`
- 원문 산출물: `archive/raw/<host>/<YYYY-MM-DD>/...`
- 정제 산출물: `data/processed/조례/<지자체>/<지자체>-<카테고리>-<조례명>.md`
- 보조 산출물: 필요 시 `archive/processed/.../source.md`에 인용 메타 누적

## 워크플로우

### 1. 수집 목표 결정

먼저 사용자의 목표가 무엇인지 정한다.

- **최신 시행 조례 목록**이 목표면 `ELIS`를 먼저 본다.
- **최근 제정·개정 흐름**이 목표면 `지방의회 의안검색`을 먼저 본다.
- **최종 본문 검증**이 목표면 `law.go.kr`를 반드시 포함한다.
- **상충 정보 해소**가 목표면 `ELIS → 의회 → law.go.kr` 순으로 대조한다.

### 2. 자료원 선택 규칙

아래 우선순위를 기본으로 한다.

1. `elis.go.kr`
   - 최신 유효 조례 목록, 공포번호, 시행일, 소관부서, 개정 이력의 1차 출처
2. 해당 지자체 의회 포털
   - 최신 조례안·규칙안, 발의자, 소관위원회, 처리결과의 2차 출처
   - 아직 ELIS에 반영되지 않은 최근 입법 동향을 확인할 때 사용
3. `law.go.kr`
   - 최종 본문과 연계 법령 검증용 3차 출처

판단 규칙:

- ELIS와 의회 정보가 다르면 **시행 여부는 ELIS를 우선**한다.
- 의회에만 있고 ELIS에 없으면 **계류 또는 반영 전 조례안** 가능성을 먼저 검토한다.
- law.go.kr 본문이 있으면 최종 본문 검증에 사용하되, 공포·시행 메타는 ELIS를 우선 참고한다.

### 3. 원문 보존

- 원문은 반드시 먼저 `archive/raw/`에 보존한다.
- `*.go.kr`는 `gokr-fetch` 또는 `gov-archive` 계열 도구를 우선 사용한다.
- 지방의회처럼 `go.kr`가 아닌 보조 출처는 사용자가 명시적으로 요청했거나, 최신 조례안 확인에 꼭 필요한 경우에만 사용한다.
- 원문 저장 시 가능한 한 본문 파일과 `.meta.json`을 함께 남긴다.
- 최소 메타 필드:
  - `source_url`
  - `collected_at`
  - `content_sha256`
  - `saved_file`
- 추출 대상:
  - 조례 명칭
  - 본문
  - 시행일
  - 공포번호
  - 소관부서
  - 개정 연혁
  - 의회 처리 정보(해당 시)

### 4. 메타데이터 추출과 교차검증

반드시 아래 필드를 채운다.

- `시행일`
- `소관부서`
- `공포번호`

검증 순서:

1. ELIS에서 시행일, 공포번호, 개정 이력을 읽는다.
2. 의회 의안검색에서 발의/처리 상태와 소관위원회를 읽는다.
3. law.go.kr 본문과 조문 구성을 대조한다.
4. 필드가 비거나 충돌하면 저장 전에 한 번 더 교차 확인한다.

저장 금지 조건:

- 시행일 또는 공포번호가 비어 있는데 보조 출처로도 채우지 못한 경우
- 의회 의안만 있고 실제 공포 여부를 확인하지 못한 경우
- 본문 출처가 불명확한 경우

### 5. 마크다운 가공

- 파일 상단을 HTML 주석(`<!-- -->`)으로 감싸 메타데이터를 둔다(렌더링 시 숨김, grep 가능).
- 본문은 원문 텍스트를 **토씨 하나 바꾸지 않고 그대로 보존**한다.
- 조문 구조를 나눌 때도 원문 문구를 고치지 않는다.
- 제목, 공포문, 부칙, 별표가 있으면 누락 없이 포함한다.
- PII가 등장할 경우 `pii-mask` 스킬을 통과시킨 결과만 저장.

### 6. 주제별(Semantic) 분류

본문을 분석해 아래 카테고리 중 **하나**만 선택:

- `일반행정` — 시정 운영, 조직, 인사, 재정, 자산
- `보건복지` — 복지 서비스, 보건소, 노인·아동·장애인 지원
- `교통안전` — 도로, 교통망, 재난·안전, 소방, 보행
- `산업경제` — 지역경제, 일자리, 기업 지원, 상업, 농업
- `도시환경` — 도시계획, 주택, 환경 보호, 공원, 청소
- `교육문화` — 교육 지원, 도서관, 예술, 관광, 체육

분류 규칙:

- 제목보다 **제1조 목적**을 우선한다.
- 소관부서는 보조 근거로만 사용한다.
- 둘 이상의 분야에 걸치면 실제 규율 대상이 가장 큰 쪽 하나만 선택한다.
- 분류가 모호하면 사용자에게 확인을 요청한다(임의 결정 금지).

### 7. 저장 및 누적 (Incremental Accumulation)

- 경로: `data/processed/조례/<지자체>/<지자체>-<카테고리>-<조례명>.md`
  (사용자가 다른 경로를 명시하면 그 경로 우선)
- 파일명 규칙:
  - 기본 형식은 `<지자체>-<카테고리>-<조례명>.md`
  - 괄호, 따옴표, 슬래시, 콜론 등 특수문자는 제거한다.
  - 공백은 유지 가능하되, 파일시스템 충돌 문자는 남기지 않는다.
  - `「」`, `『』`는 제거한다.
- **개정·변경 시 기존 내용을 덮어쓰지 않는다.** 파일 하단 `## 개정 이력` 섹션에 새 항목을 append.
- 동일 공포번호가 이미 있으면 중복 추가하지 않고 보고만 한다.

갱신 규칙:

- 기존 파일이 있으면 최신 공포번호가 이미 반영됐는지 먼저 확인한다.
- 새 개정이 확인되면 본문 전체를 새로 검증한 뒤 `## 개정 이력`에 항목을 추가한다.
- 조례안 단계 자료는 시행 조례 파일에 섞지 않고, 필요하면 별도 메모로 남긴다.

## 마크다운 표준 양식

```markdown
<!--
[요약 설명]
{조례의 목적과 주요 내용을 1~2문장으로 요약}

[법규 메타데이터]
- 지자체: {지자체명}
- 소관부서: {부서명}
- 시행일: {YYYY-MM-DD}
- 공포번호: {제0000호}
- 원본 URL: {URL}
- 수집시각: {ISO-8601}
- sha256: {전체해시 또는 짧은해시}
- 태그: {키워드1, 키워드2, ...}
-->

# {조례 명칭}

{원문 텍스트...}

## 개정 이력

- {YYYY-MM-DD} 공포 {제0000호}: {개정 요지}
  - 출처: {원본 URL} · sha256:{짧은해시}
```

## 완료 체크리스트

- [ ] 원본이 `archive/raw/`에 보존되고 SHA-256이 기록되었는가
- [ ] ELIS, 의회, law.go.kr 중 필요한 출처가 모두 교차 확인되었는가
- [ ] 메타데이터 주석에 지자체·소관부서·시행일·공포번호가 모두 채워졌는가
- [ ] 카테고리가 6종 중 하나로 결정되었는가(임의 신설 금지)
- [ ] 파일명이 `<지자체>-<카테고리>-<조례명>.md` 규칙을 따르는가
- [ ] 기존 파일이 있을 경우 덮어쓰기가 아니라 `## 개정 이력`에 append 했는가
- [ ] PII 마스킹·출처 footer가 포함되었는가

## 안전·정책

- 외부 호출은 `*.go.kr` 화이트리스트를 우선. 그 외 도메인은 명시 동의 후 사용.
- robots.txt·rate limit 준수. 위반 의심 시 중단하고 사용자에 보고.
- 정치적 해석·평가는 본문에 섞지 않는다(사실 카드만 작성).
- 최신 조례 수집에서 지방의회 포털을 쓰더라도, **시행 여부를 의회 자료만으로 확정하지 않는다.**

## 성남시 작업에서 확인된 실전 패턴

- 성남시는 `ELIS → 성남시의회 의안검색 → law.go.kr` 순으로 확인하면 최신성과 정확성을 함께 확보하기 좋다.
- 성남시의회는 최신 조례안 확인에는 유용하지만, 최종 시행 조례 판정은 ELIS와 law.go.kr로 다시 확인해야 한다.
- 의회 포털 원문도 `archive/raw/www.sncouncil.go.kr/<date>/...` 아래 HTML과 `.meta.json`으로 보존할 수 있다.
- Semantic 저장 구조는 행정조직명이 아니라 조례 목적과 규율 대상을 기준으로 정하는 편이 검색성과 재사용성이 높다.
