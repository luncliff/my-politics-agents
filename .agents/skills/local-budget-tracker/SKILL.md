# local-budget-tracker

대한민국 지방자치단체의 사업별 예산/결산을 **사업 단위 메모리 파일**로 점진적으로 누적·갱신한다.
사업 코드·부서·**세출예산 성질별 분류** 변동을 시계열로 추적해 정책 집행 형태의 진화를 가시화한다.

## 대상 지자체 식별

- 작업 시작 시 저장소 루트의 [`location.txt`](../../../location.txt)를 먼저 읽어 **대상 행정구역**을 확정한다.
- 본 스킬에서 `<지자체>`는 기초자치단체(시·군·구), `<광역>`은 광역자치단체(도·특별시·광역시)를 의미한다.
- 사용자가 채팅에서 다른 지자체를 명시하면 그 지시가 `location.txt`보다 우선한다.

## 핵심 원칙

- **사업 = 파일 1개.** 사업명 한글 그대로(띄어쓰기 없이) 파일 1개를 만든다.
- **메모리 우선.** 작업 재개 시 항상 해당 파일을 먼저 읽고 누락 연도부터 채운다.
- **편성목 변경 = 시그널.** 성질별 분류가 바뀌면 사업 형태가 바뀐 것으로 간주하고 사유를 분석한다.
- 사실 카드(Facts)와 해석(Interpretation)을 분리한다(중립 원칙).

## 워크플로우

### Phase 0. 사업 메모리 파일 초기화

조사 대상이 정해지면 즉시 전용 파일을 생성한다.

- **경로:** `archive/processed/budgets/<사업명>.md`
  - 한글 사업명을 띄어쓰기 없이 사용. 특수문자만 `_`로 치환.
  - 필요하면 지자체 접두어를 붙여 충돌 방지: `<지자체>_<사업명>.md`.
- 파일이 이미 있으면 새로 만들지 않고 **읽고 이어 작성**.

초기 템플릿 → [./assets/budget-memory-template.md](./assets/budget-memory-template.md)
(자산이 없으면 아래 "메모리 파일 템플릿"을 그대로 복사)

### Phase 1. 자료 수집 (편성목 중심)

연도별 예산서·사업설명서에서 다음을 추출한다:

- 담당 부서 / 세부사업코드 / 사업명 변경 여부
- **세출예산 성질별 분류(편성목)**: 인건비, 일반운영비, 사무관리비, 일반보전금, 민간이전, **민간위탁금**, **민간경상보조**, **출연금**, 자본지출 등
- 예산액(본예산·추경 합계 명시)

소스 우선순위:
1. **대상 지자체 본청** 정보공개 > 재정공시 (세입·세출 예산서, 결산서, 사업설명서)
2. **지방재정365** (`lofin.mois.go.kr`) — 사업코드 시계열 보완용(전국 공통)
3. **대상 지자체 의회** 의안정보·회의록 (해당 시·군·구 의회 포털)
4. 위 소스를 `gokr-fetch` 스킬로 `archive/raw/`에 보존(필수, SHA-256 기록).

실제 도메인은 `location.txt`로 식별된 지자체에 맞춰 결정한다.

### Phase 2. 사업 형태 변화 추적 (Morphological Tracking)

전년 대비 **편성목**이 바뀌면 표의 `형태 변경 사유 및 분석` 칸에 다음을 기입:

| 패턴 (전 → 후) | 추정 의미 |
| --- | --- |
| 일반운영비/사무관리비 → 민간위탁금 | 직영 → 민간 위탁 |
| 일반운영비 → 출연금 | 직영 → 산하기관/출연기관 이관 |
| 민간경상보조 → 민간위탁금 | 보조사업 → 위탁사업 전환 |
| 자본지출 신규 편성 | 시설 조성 단계 진입 |
| 사업코드 통폐합/분할 | 사업 구조 재편 |

추정은 반드시 `(추정)` 표기하고, 회의록·검토보고서로 근거를 보강한다(Phase 3).

### Phase 3. 집행 추적 및 교차 검증

- 결산서에서 예산액·집행액·집행률·**불용액·이월액**과 사유를 기록.
- 형태가 바뀐 연도는 대상 지자체 의회의 **예결특위/소관 상임위 검토보고서·회의록**을 검색해 다음을 확인:
  - 위탁기관 선정 지연, 협약 미체결 등 행정 병목
  - 의원 지적사항(중복사업, 효과성, 위탁 적정성 등)
- 정책 리스크 섹션에 사실만 기재.

### Phase 4. 갱신 루프 (Iteration)

- 매 턴 종료 시 **해당 사업 파일을 최신 상태로 저장**하고 `진행 로그`를 갱신한다.
- 자료가 막히면 Phase 1로 돌아가 보조 출처(지방재정365·회의록)로 우회 확보.
- 모든 연도·항목이 채워지면 `현재 단계: 완료(Completed)`로 표기하고 사업 형태 변화 요약을 작성.

## 완료 체크리스트

- [ ] 파일 경로가 `archive/processed/budgets/<한글사업명>.md`로 정확한가
- [ ] 작업 재개 시 기존 파일을 먼저 읽고 누락 연도부터 채웠는가
- [ ] **연도별 편성목**이 모두 채워졌는가(빈 칸은 `n/a`가 아니라 `진행 로그`에 To-Do로 등록)
- [ ] 편성목이 바뀐 모든 연도에 변경 사유 분석이 있는가(추정은 `(추정)` 표기)
- [ ] 원본이 `archive/raw/`에 보존되고 SHA-256이 5번 섹션에 기록되었는가
- [ ] 의회 지적사항이 발화 시점·위원회·링크와 함께 기록되었는가
- [ ] 정치적 해석·평가가 본문에 섞이지 않았는가

## 안전·정책

- 외부 호출은 `*.go.kr` 화이트리스트 우선. 신규 도메인은 명시 동의 후 사용.
- robots.txt·rate limit 준수.
- 회의록에서 추출한 인명·연락처는 `pii-mask` 통과 후에만 저장.
