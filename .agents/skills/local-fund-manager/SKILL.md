# local-fund-manager

대한민국 지방자치단체 기금(Funds) 운용 현황을 **메모리 기반 조사 방식(Memory-driven Research)**으로 누적 관리한다.
중복 검색을 막고 갱신이 필요한 항목만 타겟 조사한다.

## 대상 지자체 식별

- 작업 시작 시 저장소 루트의 `location.txt`를 먼저 읽어 **대상 행정구역**을 확정한다.
- 본 스킬에서 `<지자체>`는 대상 지방자치단체, `<생활권>`은 해당 지자체 내부 지역 구분을 뜻한다.
- 사용자가 채팅에서 다른 지자체를 명시하면 그 지시가 `location.txt`보다 우선한다.

## 워크플로우

### 1. 메모리 인덱스 우선 (Memory-first)

작업 시작 시 반드시 메모리 파일을 먼저 읽는다.

- **경로:** `data/processed/funds/00_Fund_Memory_Index.md`
- 메모리 파일에서 다음을 파악한다:
  - 각 기금별 `Last Updated`(최종 갱신 일자)
  - `Missing Info`(부족한 항목: 조성액·근거 조례·집행 사업 등)
  - `Next Review`(차기 갱신 트리거: 예산안 공고, 조례 개정 등)
- 메모리 파일이 없으면 `data/processed/funds/*.md`를 스캔해 **초기 인덱스를 생성**한 뒤 진행.
- **금지:** 메모리 확인 없이 광범위 웹 검색·전체 디렉터리 재스캔 수행.

### 2. 갱신 대상 식별 → 타겟 조사

메모리에서 도출된 대상만 조사한다. 일반 검색이 아닌 **구체 키워드**로 타겟팅:

- 좋은 예: `"2026 <지자체> 통합재정안정화기금 운용계획"`, `"<지자체> 도시개발특별회계 2025 결산"`
- 나쁜 예: `"<지자체> 기금"`

**조사 도구·소스:**

- `grep_search`로 워크스페이스 내 기존 자료 우선 확인.
- `gokr-fetch` 스킬로 대상 지자체 본청 재정정보공개 포털·예산서·기금운용계획 원본 보존.
- 근거 조례 확인은 `local-ordinance-processor` 산출물 또는 `legal-data-retrieval` 스킬 호출.
- 지방의회 논의는 대상 지자체 의회 회의록(예산결산특별위원회 등) 검색.
- 가능하면 **병렬 호출**로 단축.

### 3. 개별 기금 마크다운 표준 양식

경로: `data/processed/funds/<기금이름>.md` (공백·특수문자는 `_`로 치환)


### 4. 메모리 인덱스 갱신 (필수 마지막 단계)

개별 파일을 갱신했으면 **반드시** 메모리 인덱스의 해당 행을 갱신한다.

```markdown
| 기금명 | 파일 | Last Updated | Missing Info | Next Review |
| --- | --- | --- | --- | --- |
| 통합재정안정화기금 | [link](./통합재정안정화기금.md) | 2026-04-30 | 2026 결산 | 2027-03 결산공고 후 |
```

### 5. 지역 태깅 규칙

지역 연관성(섹션 4)은 **사실 기반**으로만 작성한다.

- 사업비 일부라도 해당 동에 집행되거나, 사업 대상지가 해당 동을 포함할 때만 기재.
- 행정구역 인접·간접효과 추정은 별도 표시(`(추정)`)하고 근거 명시.
- 정치적 해석·평가는 금지(중립 원칙).

## 완료 체크리스트

- [ ] 작업 시작 시 메모리 인덱스를 먼저 읽었는가
- [ ] 갱신 대상이 메모리에서 명확히 식별되었는가(임의 전체 재조사 금지)
- [ ] 원본이 `archive/raw/`에 보존되고 SHA-256이 footer에 기록되었는가
- [ ] 5개 섹션(기본정보·예산·집행사업·지역연관성·특이사항)이 모두 채워졌거나 결측은 메모리의 `Missing Info`에 반영되었는가
- [ ] 메모리 인덱스의 `Last Updated` / `Next Review`가 갱신되었는가
- [ ] PII가 없고 정치적 해석이 본문에 섞이지 않았는가

## 안전·정책

- 외부 호출은 `*.go.kr` 화이트리스트 우선. 신규 도메인은 명시 동의 후 사용.
- robots.txt·rate limit 준수.
- 산출물은 사실 카드(Facts)와 해석(Interpretation)을 분리한다.
