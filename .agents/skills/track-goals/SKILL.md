# track-goals

세션 시작 시 목표를 구조화하고, 작업 중 진행 상태를 갱신하며, 세션 종료 시 잔여 항목을 기록한다.

## 입력

- `$ARGUMENTS`: 목표 설명 (자연어). 생략 시 이전 세션 잔여 목표를 불러온다.

## 상태 파일

- 경로: `.goals/current.md`
- 이 파일은 gitignore 대상이며 로컬 세션 상태 전용.

## 절차

### 1. 목표 선언 (세션 시작)

인자가 있으면 새 목표를 등록한다. 없으면 `.goals/current.md`를 읽어 잔여 목표를 표시한다.

목표를 하위 단계로 분해한다:
- 각 단계에 적합한 agent/skill을 매핑
- 예상 산출물을 명시

### 2. 진행 갱신 (작업 중)

주요 단계 완료 시 `.goals/current.md`를 갱신:
- `[ ]` → `[x]` 체크
- 완료 시각 기록
- 예상치 못한 하위 작업 추가 시 사유 명시

### 3. 세션 종료 정리

미완료 항목을 `.goals/current.md`에 잔류시킨다.
완료된 목표는 `## 완료 이력` 섹션으로 이동 (날짜 + 한 줄 요약).

## 상태 파일 형식

```markdown
# Session Goals

updated: YYYY-MM-DD HH:MM

## 미완료

- [ ] <목표 1>
  - [ ] 단계 A — agent: <agent명>
  - [ ] 단계 B — skill: <skill명>

## 완료 이력

- YYYY-MM-DD: <완료된 목표 한 줄 요약>
```

## 제약

- `.goals/current.md`에는 PII를 기록하지 않는다.
- 목표 단계에 agent/skill을 매핑할 때 존재하지 않는 정의를 참조하지 않는다.
- 이 스킬은 계획을 세울 뿐 직접 실행하지 않는다. 실행은 사용자 승인 후 해당 agent/skill이 담당.
