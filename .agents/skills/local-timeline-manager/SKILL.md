# local-timeline-manager

대한민국 지방자치단체 행정·입법 타임라인을 **연도별 누적 마크다운**으로 유지한다.
상위 광역자치단체 일정과의 시차를 함께 추적해 보조재원 매칭이 큰 예산 흐름을 가시화한다.

## 대상 지자체 식별

- 작업 시작 시 저장소 루트의 `location.txt`를 먼저 읽어 **대상 행정구역**을 확정한다.
- 본 스킬에서 `<지자체>`는 대상 기초/광역 자치단체, `<상위광역>`은 그 상위 광역자치단체를 의미한다.
- 사용자가 채팅에서 다른 지자체를 명시하면 그 지시가 `location.txt`보다 우선한다.

## 핵심 원칙

- **Append-only.** 기존 행을 지우거나 덮어쓰지 않는다. 새 일정은 시기에 맞는 위치에 추가.
- **출처 링크 의무.** 모든 행은 공식 공고·의사일정 URL을 `[text](url)`로 포함.
- **용어 엄격.** "가내시" ≠ "확정내시". 본예산 ≠ 추경. 본회의 ≠ 상임위.
- **공고 기반 우선.** 추측 일정은 별도 "검토 필요" 섹션에만 기재.
- 사실 카드(Facts)와 해석(Interpretation)을 분리한다(중립 원칙).

## 워크플로우

### 1. 정보원

우선순위:
1. **대상 지자체 의회** — 의사일정, 행정사무감사 계획서, 회의록
2. **대상 지자체 본청** — 본예산 편성 지침, 주민참여예산 운영계획, 재정공시
3. **상위 광역자치단체 본청** — 예산 편성 지침, **가내시·확정내시** 통보 일정
4. **상위 광역자치단체 의회** — 광역 의사일정(대상 지자체 예산 심의의 상위 일정)

원본은 `gokr-fetch` 스킬로 `archive/raw/`에 보존(SHA-256 기록 필수).
조사 단계는 `@researcher-kr-website` 에이전트 활용 권장.

### 2. 추출 항목

- **지방의회**: 본회의 회기, 상임위(소관 식별), 예결특위, 행정사무감사 기간
- **지방행정**: 본예산 편성 지침 시달, 주민참여예산 공모·심의·총회, 추경 편성, 결산 공고
- **상위 광역단체**: 본예산 편성 지침, **광역보조금/도비 가내시 통보일**, **확정내시 통보일**, 광역 추경 일정
- **상임위 매핑**: 각 안건·예산 항목의 소관 상임위를 식별

### 3. 광역-기초 시차 분석

대상 지자체 예산 일정 옆에 관련된 상위 광역자치단체 일정을 함께 기록해 시차를 드러낸다.

| 도 일정 | 시 영향 | 추적 방법 |
| --- | --- | --- |
| 도 본예산 편성 지침 | 시 편성 지침 후속 | 일자 차이를 비고에 기재 |
| 도비 **가내시** | 시 부서 사업 추정치 반영 | 가내시일 ↔ 시 본예산 편성 시작일 |
| 도비 **확정내시** | 시 본예산 최종 조정 | 확정내시일 ↔ 시 의회 제출일 |
| 도 추경 확정 | 시 추경 매칭 | 도 추경 의결일 ↔ 시 추경 의결일 |

### 4. 저장 및 누적

- **경로:** `archive/processed/timeline/<지자체>_타임라인_<YYYY>.md` (연도별 1파일)
  - 사용자가 다른 경로를 명시하면 그 경로 우선.
- 표는 시기 오름차순 유지. 새 일정은 해당 월·분기 위치에 **insert**(append-only 원칙).
- 동일 일정이 이미 있으면 중복 추가 금지(상태 변경은 같은 행의 `비고`에 추기).
- 파일이 너무 커지면(>800행 권장 임계) 분기별 분할(`..._<YYYY>_Q1.md` 등) 후 인덱스 파일 추가.

### 6. 갱신 루프

매 턴 종료 시:
1. 새로 발견한 일정을 **확정/검토필요** 중 올바른 표에 추가.
2. 기존 "검토 필요" 항목이 공고로 확정되면 **확정 표로 이동**(원래 행 삭제 가능, 단 비고에 `(검토필요→확정 YYYY-MM-DD)` 기록).
3. `마지막 갱신` 메타 갱신.
4. 새로운 패턴(예: 도 가내시가 예년보다 한 달 늦음)은 retrospective 후보로 메모.

## 완료 체크리스트

- [ ] 모든 행에 공식 출처 링크가 있는가
- [ ] 가내시/확정내시, 본예산/추경, 본회의/상임위 용어가 정확한가
- [ ] 상임위 소관이 식별되어 비고 또는 구분 칸에 기재되었는가
- [ ] 도 일정(가내시·확정내시·도 추경)이 시 일정과 함께 추적되는가
- [ ] 추정 일정이 "검토 필요" 섹션에만 있는가(확정 표 오염 금지)
- [ ] 기존 행을 지우지 않고 append/insert만 했는가
- [ ] 원본이 `archive/raw/`에 보존되고 SHA-256이 출처 섹션에 있는가

## 안전·정책

- 외부 호출은 `*.go.kr` 화이트리스트 우선.
