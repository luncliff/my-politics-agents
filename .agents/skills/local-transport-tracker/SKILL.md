# local-transport-tracker

대한민국 지방자치단체의 버스 교통정보를 공식 출처에서 수집·집계하고, 인용 메타가 포함된 보고서를 생성한다.

## 언제 쓰는가

- 특정 지역의 버스 이용량·정류장 수·배차간격 현황을 정량화할 때
- 광역버스·공항버스·지역버스 노선 목록과 경로를 확인할 때
- 버스 노선 신설·변경·폐지 거버넌스(기초지자체/광역지자체/국토부/의회 역할)를 분석할 때
- 준공영제·반공영제 운영 구조를 파악할 때
- 환승센터·BRT·교통거점 계획 자료를 수집할 때

## 대상 지역 식별

- 작업 시작 시 저장소 루트의 [`location.txt`](../../../location.txt)를 먼저 읽어 기준 행정구역을 확정한다.
- 사용자가 다른 지역을 명시하면 그 지시가 `location.txt`보다 우선한다.

## 자료원 우선순위

| 순위 | 출처 | 수집 내용 |
|---|---|---|
| 1 | `gits.gg.go.kr` — 경기도교통정보센터(GITS) 원시데이터 | 지역별 버스이용객수(연간), 통행구분별 환승 분석, BIT 설치 현황 |
| 2 | `www.gbis.go.kr` — 경기버스정보시스템 | 광역버스·공항버스 노선 목록, 정류장별 배차, 실시간 정보 |
| 3 | `www.data.go.kr` — 공공데이터포털 | 지자체별 버스정류장·노선·BIT·공항버스 CSV/XLSX |
| 4 | 해당 지자체 공식 사이트(`.go.kr`) | 공항버스 운행 일정, 준공영제 계획, 노선 공고 |
| 5 | `www.law.go.kr` | 여객자동차운수사업법·대중교통육성법·수도권교통특별법 조문(거버넌스 분석 시) |

## 워크플로우

### 1. 수집 목표 결정

사용자 요청을 아래 4가지 모드 중 하나로 분류:

- **이용량 모드**: GITS 원시데이터 → 지역별 월간 승하차 합산, 피크 시간대, 환승 비중
- **노선·배차 모드**: data.go.kr CSV → 노선명·배차간격·경유 정류장 목록
- **공항버스 모드**: GBIS + 지자체 공식 XLSX → 공항버스 노선·정류장·운행 시간
- **거버넌스 모드**: law.go.kr + 지자체 공고 → 신설·변경·폐지 절차, 권한 주체 분석

전체(`전체` 인수 또는 미지정)는 4가지 모드를 순서대로 수행.

### 2. 원문 보존

- 수집 즉시 `archive/raw/<host>/<basename>`에 저장.
- `.meta.json` 필수 필드:
  ```json
  {
    "source_url": "...",
    "collected_at": "<ISO-8601 KST>",
    "content_sha256": "...",
    "saved_file": "..."
  }
  ```
- 파일명에 수집일 접두어 금지 — 시간 추적은 `.meta.json`의 `collected_at`으로.

### 3. 데이터 집계 방법

**이용량 (GITS XLSX 기준)**:
- 행정동 목록을 확정한 후 해당 행을 필터링.
- 평일/주말 평균과 범위(min~max)를 각각 산출.
- 피크 시간대(가장 높은 시간대 1~2개)를 명시.
- 환승 비중 = 환승통행 / 전체통행 × 100.

**노선·배차 (CSV 기준)**:
- 정류장 CSV의 행정동 컬럼으로 지역 필터.
- 고유 노선명 수와 배차간격(평일) 분포(≤10분 / 11-20분 / 21-30분 / 31분 이상) 계산.
- 배차간격 최소·최대·중앙값 명시.

**공항버스 (XLSX 기준)**:
- 시트별 노선명 목록 추출.
- 각 시트에서 조사 지역 내 정류장명 필터.

### 4. 거버넌스 분석 (선택)

버스 노선 관리 권한 구조는 아래 법령 조문을 기준으로 기술:

| 법령 | 핵심 조문 |
|---|---|
| 여객자동차운수사업법 | 제84조(노선 인가), 제86조(운행계획 변경) |
| 대중교통육성법 | 제7조(광역 교통계획) |
| 수도권대중교통특별법 | 제3조(협의체), 제6조(노선 조정) |
| 지방자치법 | 제13조(자치사무), 제27조(조례 제정) |

권한 주체별 정리 형식:

```
- 기초지자체(시청): 관할 내 시내버스 노선 인가·변경·폐지 신청, 준공영제 계약
- 광역지자체(도청): 광역버스 노선 조정, 준공영제 운영비 보조, 환승센터 계획
- 국토교통부: 면허 기준 고시, 공항버스 노선 인가 최종 권한
- 지방의회(시의회/도의회): 예산 심의, 조례 제정·개정, 행정사무감사
```

### 5. 보고서 생성

산출물 경로: `archive/processed/YYYY-MM <지역명> 버스 <주제>.md`

파일 상단 YAML frontmatter:

```yaml
---
title: "<지역명> 버스 <주제> 브리핑"
source_url: "<대표 수집 URL>"
collected_at: "<ISO-8601>"
content_sha256: "<hash>"
license: "공공데이터 이용허락 / KOGL Type 1"
pii_masked: false
---
```

보고서 구성 (모드별 선택):

```markdown
# <지역명> 버스 <주제> 브리핑

기준 YYYY-MM, 대표 출처 ...

## 핵심 수치          ← 이용량 모드
## 노선·배차 현황      ← 노선 모드
## 공항버스 노선       ← 공항버스 모드
## 수집 방법          ← 항상 포함
## 한계·주의사항       ← 항상 포함
## 거버넌스 구조       ← 거버넌스 모드

---
출처: <URL> · 수집 <ISO-8601> · sha256:<short>
```

## 파일명 규칙

- 형식: `YYYY-MM <광역> <기초> 버스 <주제>.md`
- 예시:
  - `2026-04 경기도 성남시 분당구 버스 교통정보 이용량.md`
  - `2026-05 경기도 성남시 버스 노선 관리 권한 구조 브리핑.md`
  - `2026-05 경기도 성남시 환승센터 서부권 교통사업 재검증.md`

## 완료 체크리스트

- [ ] `location.txt` 또는 사용자 지정으로 대상 지역 확정
- [ ] 원본이 `archive/raw/`에 보존되고 SHA-256 기록
- [ ] 집계 수치에 출처 URL이 인라인으로 포함됨
- [ ] 보고서가 `archive/processed/`(flat) 에 저장됨 (subfolder 금지)
- [ ] YAML frontmatter의 `source_url`, `collected_at`, `content_sha256` 모두 채워짐
- [ ] PII 없음 확인 (`pii_masked: false`) 또는 마스킹 처리

## 안전·정책

- `*.go.kr` 화이트리스트 우선. 그 외 도메인(예: gbis.go.kr)은 명시 동의 후 사용.
- GITS raw ZIP/XLSX는 직접 CLI 다운로드 불가한 경우 수동 사전 다운로드 후 경로 참조.
- GBIS 실시간 API는 공공데이터포털 인증키 없이 자동 수집 불가 → 인증키 확보 여부를 먼저 확인.
- robots.txt·rate limit 준수. 위반 의심 시 즉시 중단하고 사용자에 보고.
- 수치 해석과 정책 평가는 분리 — 사실 카드에는 수치와 출처만 기입.

## 성남시 작업에서 확인된 실전 패턴

- GITS는 `data/list.do` 게시판에서 연도별 ZIP/XLSX를 직접 다운로드. URL 패턴: `download2.do?f=<encoded>`.
- data.go.kr CSV에서 행정동 필터링 시 `행정동` 또는 `정류장명` 컬럼의 접두어로 분당구 동 이름 확인.
- 공항버스 XLSX는 시트별로 노선이 구분됨. 지역 내 정류장은 `판교`, `운중`, `정자`, `미금` 등 키워드로 필터.
- 준공영제 운영비 분담 비율과 적자 보전 구조는 경기도 공식 발표 자료(`gnews.gg.go.kr`)를 1차 출처로 사용.
