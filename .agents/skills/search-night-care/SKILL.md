# search-night-care

대한민국 특정 지역의 심야·새벽 의료 접근성을 조사해 Markdown 보고서로 정리한다.

## 대상 지역 식별

- 작업 시작 시 저장소 루트의 `location.txt`를 먼저 읽어 대상 지역을 확정한다.
- 사용자가 채팅에서 다른 지역을 명시하면 그 지시가 `location.txt`보다 우선한다.
- `location.txt`에 시·군·구까지 있으면 그 범위를 기본 조사 단위로 사용한다.

## 목표

- 대상 지역 병원·의원과 응급실을 함께 조사한다.
- 소아청소년과와 가정의학과를 우선 확인한다.
- 기본 판정 시간은 `21:00-06:00`으로 둔다. 사용자가 다른 기준을 주면 그 기준을 우선한다.
- 병원별 주소, 전화번호, 진료시간, 진료과목, 야간 가능 여부를 함께 정리한다.

## 우선 소스

1. E-GEN 통합홈페이지: 일반 병원, 응급실, 달빛어린이병원
2. 병원 공식 홈페이지: 야간 진료 공지, 외래 운영시간 교차확인
3. 보조 확인용 공식 페이지: 병원 소개, 진료안내, 오시는 길

## 핵심 원칙

- 신규 도메인은 먼저 확인하고 필요한 경우에만 추가한다.
- 진료시간은 페이지 노출값을 우선하며, 전화 확인이 필요한 경우 `확인 필요`로 남긴다.
- 응급실 운영과 외래 야간진료는 분리해서 판정한다.
- 위치 정보는 주소를 기본값으로 쓰고, 좌표가 있으면 보조값으로 둔다.
- `야간 가능`, `응급 대안 전용`, `야간 불가`, `확인 필요` 네 범주를 유지한다.

## 워크플로우

### 1. 조사 범위 고정

`location.txt` 기준 지역명에서 조사 범위를 정한다.

- 예: `대한민국 경기도 성남시 분당구` → 분당구 전체 병원·의원 + 응급실
- 예: `대한민국 서울특별시 마포구` → 마포구 전체 병원·의원 + 응급실

### 2. 수집

1. E-GEN에서 대상 지역 일반 병원 목록을 찾는다.
2. 기관별 상세에서 주소, 전화, 기관구분, 진료시간, 진료과목을 추출한다.
3. 응급실 상세와 달빛어린이병원을 별도 수집한다.
4. 병원 공식 홈페이지로 야간 진료 문구를 교차확인한다.

### 3. 분류

- `소아청소년과` 포함 기관 표
- `가정의학과` 포함 기관 표
- `응급 대안` 표
- `야간 불가 / 확인 필요` 표

### 4. 판정

- `21:00` 이후 외래 또는 소아 야간진료가 확인되면 `야간 가능`
- 응급실만 가능하면 `응급 대안 전용`
- 운영시간이 그 이전에 끝나면 `야간 불가`
- 정보가 충돌하거나 날짜별 변동만 있으면 `확인 필요`

### 5. 저장

- 기본 경로: `archive/processed/<region-slug>-night-care/report.md`
- 출처 경로: `archive/processed/<region-slug>-night-care/source.md`
- `<region-slug>`는 `location.txt`의 지역명을 영문 또는 식별 가능한 ASCII slug로 정한다.
- 사용자가 다른 경로를 지정하면 그 경로를 우선한다.
